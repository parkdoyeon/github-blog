셀레니움과 pytest로 브라우저 테스트 하기

May 28 2019

디렉토리 구조

우선 pytest 샘플 프로젝트의 이렇다. 각각의 파일의 특징들은 아래와 같다.

D:.
│  conftest.py # 파이테스트에서 공통적으로 사용할 객체들을 구현한 모듈
│  inspection_test.py # 브라우저를 테스트하는 클래스/함수가 구현된 모듈
│  requirements.txt # pip으로 설치된 패키지 리스트
│
├─allure-results
├─bin
│      chromedriver72.exe
│      IEDriverServer.exe
├─venv
└─__pycache__

conftest.py: 공통 객체 설정하기

pytest는 conftest.py라는 이름을 가진 스크립트에서 터미널 실행시 지정할 옵션값을 설정하거나,
각 테스트 함수에서 공통으로 사용하는 오브젝트(가령 셀레니움에서 사용할 브라우저 엔진같은)를 생성할 수 있다.

import pytest
import allure

# 터미널 옵션 설정
def pytest_addoption(parser):
    # 아래에서 params 값에 세팅해줄 예정이니 패스
    # parser.addoption("--driver", action="store", default="chrome", help="Type in browser type")
    parser.addoption("--url", action="store", default="http://test.website.co.kr", help="url")

# @pytest.fixture 데코레이터를 통해 리스트 형식인 파라미터를 넘기면,
# params값이 루프를 돌면서 인자 순서대로 함수가 여러번 호출된다.
@pytest.fixture(params=["chrome", "ie"], scope="class")
def driver_get(request):
    from selenium import webdriver

    if request.param == "chrome":
        web_driver = webdriver.Chrome("bin/chromedriver72.exe")
    if request.param == "ie":
        web_driver = webdriver.Ie("bin/IEDriverServer.exe")

    request.cls.driver = web_driver
    yield # yield 예약어를 통해 web_driver사용을 일시적으로 브라우저 엔진을 반환한다.

inspection_test.py: 테스트 함수를 만드는 클래스 모듈

pytest를 실행하면 conftest.py에서 설정을 읽은 후 test가 접두어로 붙은 함수를 차례대로 호출한다.
이때, 호출 데코레이터에 파라미터를 지정해놓으면, 앞에서 설명한 conftest.py의 fixture 파라미터 처럼 값을 순차적으로 순회하면서 함수를 여러번 실행시킨다.
