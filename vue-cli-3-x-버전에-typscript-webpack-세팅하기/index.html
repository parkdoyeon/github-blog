vue cli 3.X 버전에 typscript, webpack 세팅하기

October 15, 2019 — 2 Min Read

javascripttypescriptnode

처음에 엄청나게 삽질을 하게되는데 하고나면 또 별거아닌... 것중 하나가 웹팩세팅 아닌가 싶다. 이번에 SPA도입하면서 본격적으로 vue 빌드환경을 세팅해보게 되었는데, vue.js도 그렇지만 프론트엔드 개발환경구성 관련 기술들을 처음 접해봐서 그런지 많이 헤맸다. 해보고나니 헤맨만큼 더 잘 알것같은 느낌이 들어 좋은 경험이었다.

vue cli 설치하기

vue cli는 2.X대랑 3.X대가 변화가 큰 편이다. 혹시 구버전이 있으면 프로젝트 생성 명령어부터 다르니 반드시 두 버전을 확인하고 개발해야한다.

npm i -g @vue/cli

vue cli 프로젝트 생성하기

vue create [project-name]

타입스크립트를 추가하고싶으면 프로젝트 생성시 설정단계를 통해 타입스크립트를 추가해도 되고, 생성된 프로젝트에 명령어로 typescript를 추가해도 된다.

생성된 vue 프로젝트에 typescript 추가하는 법

vue add typescript

타입스크립트 추가를 하면 main.js가 main.ts로 변경되고, 내가 설정한 옵션에 따라 이 경로에 shims-tsx.d.ts와 shims-vue.d.ts파일이 생성된다.

webpack 연동하기

vue cli는 웹팩 위에서 빌드되는 프로젝트이므로 기본 설정만으로도 당장 테스트하기에 충분하지만 그래도 webpack에서 사용하던 설정들을 몇가지 붙여야하면 vue.config.js를 통해 세팅하면 된다. 최초에 기본 생성되는 파일이 아니므로 직접 만들어야한다.

기본적으로 지원되는 옵션은 공식문서를 참조해서 작성하고, 나머지는 configureWebpack을 통해 설정한다. 해당 키값으로 설정된 옵션들은 webpack옵션에 merge되면서 빌드된다.

const path          = require('path'),
	UglifyJsPlugin  = require('uglifyjs-webpack-plugin');
	rootconfig      = require('./root.config')();

module.exports = {
	publicPath: '/',
	outputDir: path.resolve(__dirname, './dist/'+ rootconfig.buildpath),
	configureWebpack: {
		entry: {
			app : './src/' + rootconfig.buildpath + 'main.ts',
		},
		output: {
			filename: 'bundle.js'
		},
		resolve: {
			alias: {
				'@': path.resolve(__dirname, 'src/' + rootconfig.buildpath)
			}
		},
		optimization: {
			minimize: true,
			minimizer: [
				new UglifyJsPlugin({
					uglifyOptions:
					{
						sourceMap: true,
						warnings: false,
						compress: {
							unused: true
						},
						mangle: false,
						beautify: true,
						output: {
							comments: false
						}
					}
				})
			]
		}
	}
};
