Programming Elixir 6 - 문자열, 제어흐름

January 29, 2020 — 2 Min Read

elixirstringcontrol

문자열

Sigils ( symbols with magical power )

물결로 시작하는 접두어를 sigil로 부른다

~C A character list with no escaping or interpolation ~c A character list, escaped and interpolated just like a single-quoted string ~D A Date in the format yyyy-mm-dd ~N A naive (raw) DateTime in the format yyyy-mm-dd hh:mm:ss[.ddd] ~R A regular expression with no escaping or interpolation ~r A regular expression, escaped and interpolated ~S A string with no escaping or interpolation ~s A string, escaped and interpolated just like a double-quoted string ~T A Time in the format hh:mm:ss[.dddd] ~W A list of whitespace-delimited words, with no escaping or interpolation ~w A list of whitespace-delimited words, with escaping and interpolation

sigil안에서 구분자(delimiter, (, [, {, <)는 닫히지 않으면 이스케이프가 필요 없다.

Single quotations

엘릭서에서 작은 따옴표 안에 문자는 캐릭터 리스트로 판단한다.

iex> ‘cat’
‘cat'
iex> [67, 65, 84]
‘CAT'
iex> [67, 65, 84, 101]
[67, 65, 84, 101]
iex> str = ‘wombat'
‘wombat'
iex> :io.format "~w~n", [ str ] [119,111,109,98,97,116]
:ok
iex> List.to_tuple str
{119, 111, 109, 98, 97, 116}
iex> str ++ [0]
[119, 111, 109, 98, 97, 116, 0]

~n은 새줄 바꾸기를 의미한다.

~w은 언랭 언어로 스트링을 표기하는 것을 강제하는 것으로, 언랭 내에서 해당 표현이 불가능하면 그냥 반환된다.

iex(12)> '∂x/∂y'
[8706, 120, 47, 8706, 121]
iex(13)> ~w'∂x/∂y' 
["∂x/∂y”]

Binaries

바이너리는 <<>>안에 담긴 0부터 255까지의 연속된 숫자 리스트이다. 바이너리 안에 integer, float값을 담을 수 있다.

Double-Quoted Strings Are Binaries

single quotation으로 감싼 문자열은 리스트라면, double quotation으로 감싸진 string은 utf-8인코딩된 일련의 바이트 데이터다. 이렇게 정의하면 합리적인데, 다음과 같은 이유에서이다.

첫째, 반드시 글자별로 다른 메모리값을 가질 수 있어 메모리 사용이 합리적이다. 둘째, 리스트를 다루는 방식으로 문자열을 조작할 필요가 없다. 접근 방식이 편리함.

문자열 데이터를 list 모듈에서 head-tail 구조로 분리할 수도 있는데. 이 경우 아래와 같이 바이너리 바이너리데이터 형식을 빌려서 분리할 수 있다.

defmodule Utf8 do
  def each(str, func) when is_binary(str), do: _each(str, func(

  defp _each(<< head :: utf8, tail :: binary >>, func) do
    func.(head)
    _each(tail, func)
  end
  
  defp _each(<<>>, _func), do []
end

제어흐름

제어문을 사용하는 것보다 함수로 처리하는 것을 더 권장. 코드가 더 간결하고 핵심적일 수 있기 때문.

Cond vs Case cond는 여러 조건 체크가 필요한 경우 사용, case의 경우 패턴 매칭되는 요소를 찾을 때 사용

Exception 사용자 정의 예외 클래스의 경우 반드시 message를 구현해야 한다. 예외가 절대 발생하면 안되는 경우 별도로 예외처리를 별도로 하지 않지만, 그렇지 않다면 함수 끝에 느낌표를 넣으면 예외가 raise 되며, 유의미한 예외일 것이라는 관습적인 의미를 갖는다.
